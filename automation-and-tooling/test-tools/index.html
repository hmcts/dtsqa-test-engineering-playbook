Test Tools

Automation

Below are the automation tools approved by the AWG for different phases/disciplines of automated testing. Before adopting any further tools we should consider the following questions:

  • Are other project teams testing the same tech stack with an already approved existing test tool?
  • Will the tool support the programmes ability to transfer developers, testers and BAs between project teams?
  • How well supported is the tool - is it in Beta, are their support forums, how good is the documentation?
  • Are there are challenges hiring staff with the tool skillset?
  • Will the tool be easily picked up by the future support teams?
  • Is it open source or are there any costs?
  • Will the tool tie as to a particular supplier?
  • Ease of use for the team (including the existing DTS team)
  • What reporting to the business does the tool provide?

Standard test tools

The following test tools are either used independently of the build, test, deploy pipeline or can be run both as Jenkins jobs and externally.

  • Zephyr for JIRA - test management tools
  • Pa11y (run in sprint but can also be run post deploy)
  • Zap (run post deploy independent of SonarQube)
  • Sauce Labs for both automated and manual cross device testing
  • http://www.mobilephoneemulator.com/
  • Mozilla observatory for security scanning
  • SoapUI - api testing tool
  • BugMagnet
  • HTML CodeSniffer (run on demand or paste in code here http://squizlabs.github.io/HTML_CodeSniffer/)
